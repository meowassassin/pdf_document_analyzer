# 셀–주파수 기반 문서 분석기 연구 보고서 (확장판)

## 1. 연구 개요

### 1.1 연구 제목
**셀–주파수 기반 문서 분석기(Cell–Frequency Hybrid Parser)의 설계 및 구현**

### 1.2 연구 기간 및 참여 인원
- **연구 기간:** 2025.11.10 ~ 2025.12.10 (1개월)  
- **참여 인원:** 이수현 · 엄준경 · 이태준  
- **기반 기술:** Java(Spring Boot), ND4J(JTransforms), Apache PDFBox, OpenAI ChatGPT API  

### 1.3 연구 목적
본 연구의 목적은 문서를 **의미 단위 셀(Cell)** 로 세분화하고,  
이를 **주파수 영역(FFT)** 에서 분석하여 구조적 공명을 검출함으로써  
기존 토큰 기반 문서 분석보다 빠르고 효율적인 **AI 문서 이해 시스템**을 개발하는 것이다.  
또한 ChatGPT API를 활용하여 구조화된 정보를 자연어로 재구성함으로써  
문서의 핵심 요약과 의미적 해석을 자동화한다.

## 2. 연구의 필요성

현대의 문서 분석 시스템은 대부분 Transformer 기반의 LLM을 사용한다.  
이 접근은 높은 정확도를 제공하지만 다음과 같은 한계가 있다.

1. **연산 복잡도 문제:** Attention 구조는 입력 길이에 따라 O(N²)로 증가하며, 긴 문서에 비효율적이다.  
2. **비용 문제:** 긴 문서를 LLM에 직접 입력하면 토큰 사용량과 API 비용이 과도하다.  
3. **구조적 불안정성:** PDF나 보고서 등은 계층 구조가 복잡해 단순 토큰 나열로는 문서의 논리 구조를 보존하기 어렵다.

이에 본 연구는 문서를 파동적 관점(Wave-based Representation)에서 재해석하여  
언어의 리듬과 구조를 **주파수 스펙트럼**으로 표현하는 새로운 문서 해석 기술을 제안한다.

## 3. 기존 기술과 한계

| 구분 | 기존 기술 (Transformer/BERT) | 한계점 |
|------|------------------------------|---------|
| **입력 단위** | 토큰(Token) | 문맥 단절, 구조 보존 어려움 |
| **핵심 연산** | Self-Attention (O(N²)) | 긴 문서에서 비효율 |
| **문맥 처리 범위** | 짧은 문맥 (local) | 전역 구조 반영 어려움 |
| **의미 해석** | 내부 벡터 의존 | 해석 불투명 |
| **비용** | LLM 호출 의존 | 토큰 과다 사용 및 비용 증가 |

이와 달리, 본 연구는 **FFT 기반 구조 분석 + 셀 단위 문맥 모델링**을 통해  
O(N log N)의 효율적 구조 탐색을 달성하고자 한다.

## 5. 셀(Cell) 기반 문서 표현

문서의 문단, 제목, 표, 캡션 등을 의미 단위로 분리하여 셀(Semantic Cell)로 구성한다.  
각 셀은 다음과 같은 구조를 갖는다.

```text
Cell = { id, type, position, embedding_vector, layout_info }
```

**특징:**  
- 문서의 지역 문맥(local context)을 보존  
- FFT의 입력 단위로 사용되어 주파수 해석 가능  
- 문서 구조의 리듬과 패턴을 수학적으로 분석할 수 있는 최소 단위  

---

## 6. 주파수 기반 문서 구조 분석 (FFT Layer)

셀 시퀀스를 FFT(Fast Fourier Transform)로 변환하여 문서의 구조적 리듬(Structural Rhythm)을 분석한다.  

\[
Y[k] = X[k] \cdot P[k]
\]  

- \( X[k] \): 셀 시퀀스의 FFT 스펙트럼  
- \( P[k] \): 문서 구조 필터(논문형, 보고서형, 계약서형 등)  
- \( Y[k] \): 공명 결과 스펙트럼  

역변환(IFFT)을 통해 각 셀의 구조 강도(Resonance Intensity)를 구하고,  
이를 파서의 구조 점수에 결합한다.

---

## 7. 기존 구조 vs 제안 구조 비교

| 항목 | 기존 문서 분석 (Transformer/BERT) | 제안 구조 (Cell–Frequency Hybrid Parser) |
|------|------------------------------------|--------------------------------------------|
| **입력 단위** | 토큰(Token) | 셀(Cell: 의미 단위 그룹) |
| **핵심 연산** | Self-Attention (O(N²)) | FFT + Spectral Filtering (O(N log N)) |
| **문맥 유지** | 국소 문맥만 반영 | 전역 구조 패턴까지 반영 |
| **메모리 효율성** | 낮음 | 약 40~70% 절감 |
| **결과 해석** | LLM 내부 벡터 불투명 | 스펙트럼 시각화 가능 |
| **LLM 의존도** | 매우 높음 | 후처리 단계만 사용 (API 최소화) |

**결론:**  
셀–주파수 구조는 문서 분석의 핵심을 **LLM에서 수학적 구조 분석으로 이동**시켜,  
속도·정확도·비용의 균형을 달성한다.

## 8. 시스템 아키텍처

**5계층 구성**  
1. **Document Processing Layer** – PDFExtractor, LayoutAnalyzer  
2. **Semantic Cell Layer** – TokenMixer, CellBuilder  
3. **Frequency Analysis Layer** – FFTEngine, ResonanceFilterRegistry  
4. **Parsing & Validation Layer** – SpectralScoreInjector, StructureValidator  
5. **Integration Layer (ChatGPT API)** – ResultFormatter, LLMAdapter  

**데이터 흐름:**  
PDF 입력 → 텍스트 추출 → 셀 생성 → FFT 분석 → 구조 점수 결합 → ChatGPT 요약 생성

\[
R = 	ext{IFFT}(X \cdot P)
